PROJECTS

CristalWeb -MIS system

  • A centralised web management information application used by staff and students of an educational establishment. The application handled large volumes of data and was used by over 5000 people on a daily basis. Main features of the application included a payment system, registers, messaging service, timetabling, news and alerts, information management ect.
  • Used many efficient SQL queries and stored procedures to handle large volumes of data.
  • Used many automated features
  • Developed using Javascript, Jquery, Ajax, HTML, CSS, C#, asp.Net and MVC, NHibernate
  • Established application, worked on as a group to add new features and fix bugs

 

  • An application that allows bulk emails to be sent to preset or custom groups of people.  The UI was design so the user could easily create, edit and delete custom email groups by  student/staff name or ID. Emails could be created, edited, deleted, sent or resent at any time.
  • Used RabbitMQ for sending emails. The emails were treated as individual tasks and split into multiple queues.  Workers were assigned to each queue to send the emails quickly. If an email was invalid the worker would remove it from the queue and move onto the next one. The invalid email information would be stored in a log.
  • Groups were created by taking the unique ID selected by the user and queried the database using SQL to retrieve their information and assign a new group ID. A message would be added to the group.
  • Developed using Javascript, Jquery, Ajax, RabbitMQ, HTML, CSS, C#, asp.Net and MVC, NHibernate
  • Solo project, designed, developed and deployed

Automated Cafe system (Java)

  • An application that managed the running of a restaurant. The system had an interface for the staff and for the customers.
  • On the customer side, a menu and prices are displayed. Orders could be places, changed or canceled and progress would be shown.
  • On the staff side, customer orders would be displayed with a table number and time stamp.
  • The system also kept track of ingredients and would update everytime a order is places. If a specific ingredients ran out, the system would temporarily take it of the menu.
  • Developed as a group project using Java, PHP and SQL
